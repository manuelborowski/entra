Access to API

Access to the API is via GET-HTTPS requests

Authentication is required by adding, in the header, the x-api-key key.  The value should be a unique token, supplied by the owner of the site.

url : <base-url>/api/<topic>/get?fields=aa,bb&filters=aa$!$123,bb$>$456

fields and filters are optional, if not present all fields of all datasets are returned

base-url

Is the url where this page is hosted, without the trailing "/api/info"

topic

  • student : everything related to students
  • staff: everyting related to teachers and employees

fields

url to list the available fields of a topic: .../api/<topic>/fields

fields is a comma separated list of fields.

A field is a member of a dataset.  By supplying one or more fields, the data returned (per dataset) is limited to these fields.

E.g. in the example above, only fields aa and bb are returned

filters

filters is a comma separated list of filters.

If specifying more than one filter, the filters are ANDed

Template of a filter: <field><operator><value>

Field is a member of the dataset to filter on.  Note, it is not required that the field, specified at filters is also specified at fields

Operator:

  • $=$: equal to
  • $!$: not equal to
  • $<$: less than
  • $>$: more than

Value is the value to be compared with.  Note, when comparing with dates, the format is year-mont-day, e.g. 2023-10-25
